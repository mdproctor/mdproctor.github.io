Backward Chaining emerges in Drools

Drools 5.2 introduces proper backward chaining support, allowing rules to derive facts by querying for them rather than asserting them reactively. This is a significant step toward full Prolog-style logic programming within the RETE network.

To illustrate the concept, I recorded a short walkthrough of the new query syntax and how it interacts with the working memory:

▶ Watch on YouTube — Drools Backward Chaining Demo (14:32)

The query keyword in DRL lets you define named queries callable from Java or other rules:

query isContainedIn( String x, String y )
  Location( x, y; )
  or
  ( Location( x, z; ) and isContainedIn( z, y; ) )
end

rule "Find keys"
  when
    isContainedIn( "key", "office"; )
  then
    System.out.println( "key is in office" );
end

Notice that the semicolons inside the parentheses separate input from output parameters — a binding convention borrowed from constraint logic programming that took some adjustment.
