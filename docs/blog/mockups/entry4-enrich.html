# Backward Chaining emerges in Drools
date: 2011-04-18 · author: Mark Proctor · tags: Drools, BackwardChaining
Drools 5.2 introduces proper backward chaining support, allowing rules to derive facts by querying for them rather than asserting them reactively. This is a significant step toward full Prolog-style logic programming within the RETE network.
To illustrate the concept, I recorded a short walkthrough of the new query syntax and how it interacts with the working memory:

The new `query` keyword in DRL lets you define named queries that can be called from Java or from within rules themselves. Here's the essential pattern using the family relationship example from the test suite:
✦ gist inlined · gist:mproctor/3a8f72c
// Drools backward chaining query example query isContainedIn( String x, String y ) Location( x, y; ) or ( Location( x, z; ) and isContainedIn( z, y; ) ) end rule "Find keys" when isContainedIn( "key", "office"; ) then System.out.println( "key is in office" ); end
Notice that the semicolons inside the parentheses are part of the query binding syntax — this separates input from output parameters and took some getting used to coming from pure Prolog.
